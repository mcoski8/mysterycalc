## Scope for this sprint

- **In:** a nightly **sync/index** of sealed product + prices from tcgcsv into a new Supabase table
  (`sealed_products`); a `TcgCsvPriceSource` that searches that table; a `CompositePriceSource` so the search
  asks both sealed (tcgcsv index) and singles (pokemontcg.io) at once; a "Sealed" badge + type label in
  `CardSearch`; the prize row's `type` set to `sealed` when a sealed product is picked. Manual entry untouched.
- **Out / deferred:** the Vercel Cron's one-time **env setup** (`CRON_SECRET` + service-role key in the Vercel
  dashboard) — only takes effect once the app is deployed (Phase 5); the code + `vercel.json` are ready now.
- **Permanently manual:** graded (PSA/BGS/CGC) — not in TCGPlayer's catalog, so no free source.

## Design (as built)

- **Empirically-validated sealed detection (Decision 032):** a tcgcsv product is **sealed** when its
  `extendedData` has **neither a `Number` nor a `Rarity`** field. (Singles always carry `Number`; digital
  "Code Cards" carry `Rarity` but no `Number`.) Verified live across sets 1999→2026 — Jumbo Cards (333
  products) and promo sets (283) yielded **zero** false positives. **Gotcha we dodged:** adding a name-keyword
  AND-filter (Gemini's suggestion) would have wrongly dropped real sealed items with no obvious keyword
  ("2-Player Starter Set", "Collector Chest"). So field-absence is the gate; keywords only label the type.
- **One sync path, not two.** The full sync (all 217 sets, two small downloads each) measured **4.6s** — far
  under Vercel's 60s cap — so we collapsed the planned "discover vs. light-price-refresh" split into a single
  full sync used by both the script and the cron. (Avoided over-engineering; a partial-column upsert also can't
  satisfy the table's NOT-NULL columns, which the single-path full upsert sidesteps.)
- **Table** `sealed_products` (migration `20260606130000`): `product_id` PK, `name`, `clean_name`, `set_name`,
  `group_id`, `image_url`, `market_price`, `low_price`, `product_type`, `prices_updated_at`, `synced_at`.
  Public reference data — RLS on with `select using(true)`; **no** insert/update policy, so only the
  service-role sync can write. `pg_trgm` GIN index on `name` for fast `ILIKE` search.
- **Shared core** `lib/sealed/`: `classify.ts` (`isSealed` + `productType`, **pure + unit-tested**),
  `db.ts` (cookie-free `adminClient` [service-role, writes] / `readClient` [anon, reads]), `sync.ts`
  (`syncSealed()` — fetch groups → products + prices → keep priced sealed → upsert; polite UA + concurrency 6).
- **Entry points:** `scripts/sync-sealed.ts` (run via `npx tsx` for the initial populate / on-demand refresh)
  and `app/api/cron/sync-sealed/route.ts` (nightly Vercel Cron; `CRON_SECRET` bearer-auth; `maxDuration=60`).
  `vercel.json` schedules it daily at 09:00 UTC.
- **Read path:** `lib/prices/tcgcsv.ts` `TcgCsvPriceSource.search` (ILIKE every typed word against `name`,
  order by `market_price` desc, cap 12, tagged `kind:"sealed"`; returns `[]` on any trouble so singles + manual
  keep working). `lib/prices/composite.ts` `CompositePriceSource` runs sources in parallel and concatenates
  (sealed first). `getActivePriceSource()` now returns `Composite([tcgcsv, pokemontcg])`. Route unchanged.
- **UI:** `CardSearch.tsx` shows a "Sealed" badge + type label and handles the empty card-number gracefully;
  `Calculator.handleAddFromCard` sets `type: "sealed"` for sealed picks (label only — never affects the math).
  `PriceCandidate` gained a `kind: "single" | "sealed"` field.

**Landmines for next session.**
- **Sealed detection is field-absence, NOT keywords** — see Decision 032. Don't "tighten" it with a name
  keyword filter; that drops keyword-less sealed product (Collector Chest, Starter Set).
- **`tsx` is the script runner** — `npx tsx scripts/sync-sealed.ts` to repopulate/refresh sealed prices
  locally. `lib/sealed/*` use **relative** imports (not `@/`) so tsx resolves them.
- **Partial-column upsert fails** on `sealed_products` (NOT-NULL `name` etc.) — always upsert full rows. This
  is why there's a single full-sync path, not a price-only one.
- **Cron is wired but dormant until deploy.** `vercel.json` + `/api/cron/sync-sealed` exist; they only run once
  the app is on Vercel (Phase 5) AND `CRON_SECRET` + `SUPABASE_SERVICE_ROLE_KEY` are set in the Vercel project.
  Until then, data is the local populate — re-run the script to refresh.
- **Result ordering is sealed-first, by value.** A broad term ("charizard") shows sealed collections before
  single cards. Owner confirmed this is acceptable (mystery prizes are heavily sealed). Revisit if it annoys.
